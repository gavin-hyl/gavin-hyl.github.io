Real-Time Model Predictive Control Library

Control Systems C++ Optimization Robotics

As a SURF fellow at the AMBER Lab, I developed a robust, real-time model predictive control (MPC) library in C++ for robotic systems, named TORC (Trajectory Optimization for Robotic Control).

Key features of this library include:

  • Integration with CppAD for automatic differentiation, allowing efficient gradient computation
  • Implementation of the IPOPT solver for constrained nonlinear optimization
  • Support for various reduced-order dynamics models for robots
  • Flexible interfaces for defining custom cost functions and constraints
  • Real-time performance with optimized memory management and algorithm design

This library currently serves as the foundation for control systems in several robotic platforms in Professor Aaron Ames' group.

Telemetry & Data Acquisition System

Embedded Systems ESP32 WiFi Real-time Data

For Caltech Racing, I designed and implemented a telemetry and data acquisition system using an ESP32 microcontroller that enables real-time monitoring of race car performance.

The system's capabilities include:

  • Concurrent transmission of vehicle data to ground control over WiFi
  • Redundant storage of sensor data on an SD card for post-run analysis
  • Collection of critical vehicle parameters including speed, temperature, battery status, and motor performance
  • Real-time visualization interface for the race team
  • Configurable sampling rates for different sensor types

This project involved both hardware design and firmware development, providing valuable insights for vehicle optimization and troubleshooting during testing and competition.

Caltech-10 CPU Design

Computer Architecture ABEL CPLD Digital Design

As part of Professor Glen George's computer engineering course, I designed and implemented a Harvard Architecture RISC CPU in ABEL on a CPLD.

Key features of this CPU include:

  • Complete instruction set with arithmetic operations, logical operations, and data movement
  • Separate data and program memory spaces with distinct addressing modes
  • I/O handling capabilities for peripheral device communication
  • Call/return functionality with stack management
  • Pipelined architecture for improved instruction throughput

This project provided deep insights into CPU design principles, instruction set architecture, and hardware description languages. The implementation required careful consideration of timing, resource utilization, and overall system performance.

Balance Game Design & Implementation

Embedded Systems AVR Assembly SPI PCB Design

For this project, I designed and built an interactive balance game featuring an ATMega64 microcontroller with sophisticated peripheral integration.

The implementation involved:

  • Assembly of the custom PCB with all required components
  • Programming the ATMega64 with 1500+ lines of AVR assembly code
  • Interfacing with an MPU6500 inertial measurement unit through SPI protocol
  • Generating square waves on a speaker through precisely timed timer interrupts
  • Implementing LED matrix multiplexing for the game display
  • Creating robust debouncing algorithms for buttons and rotary encoder inputs

This project demonstrated my ability to work at the hardware-software interface, handling low-level programming challenges while creating an engaging user experience.
